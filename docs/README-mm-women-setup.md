## The 17 Models: How They Work in EasyML

### Standard Classifiers (8 models)
These are regular binary classifiers:
```yaml
xgb_core:
  type: xgboost
  mode: classifier
  features: [diff_seed_num, diff_win_pct, ...]
  params: {max_depth: 4, ...}
```

### Regression Models (3 models)
These predict margin, easyml auto-converts to probability via CDF:
```yaml
xgb_spread_broad:
  type: xgboost
  mode: regressor          # ← regressor mode
  features: [diff_seed_num, diff_scoring_margin, ...]
  # cdf_scale auto-fitted during training via Brier score minimization
```

### Survival Model (1 model)
This produces features for other models, excluded from final ensemble:
```yaml
survival_hazard:
  type: survival           # ← special type
  features: [seed_num, scoring_margin, ...]
  provides: ["survival_prob"]         # ← outputs predictions as feature
  include_in_ensemble: false          # ← excluded from ensemble
```

### Feature Producer Pattern
Any model can produce features via `provides`:
1. Model trains and generates predictions
2. Predictions stored as matchup-level features (e.g., `survival_prob`)
3. Other models can use `diff_survival_prob` in their feature list
4. Model excluded from final ensemble via `include_in_ensemble: false`
